ExternalControler

Par Nico


ExternalControler est un module permettant de piloter l'application par une application externe.

Il devient alors possible de controler certains module depuis un script python ou une interface web.


Les interfaces réseau à disposition sont :

  • Tcp
  • WebSocket


Paramètres de configuration :

Le choix des paramètres réseau (interface, port) se fait depuis le fichier de configuration EEPROM :

  • server_type : choix du type d'interface pour le serveur. Les valeurs possibles sont : tcp / websocket (insensible à la casse)
  • port : numéro du port de communication


Description de l'interface de contrôle :

Une commande peut être consituée d'une ou plusieurs instructions.
Dans le cas d'une commande à plusieurs instructions, les instructions sont séparées par un retour chariot (à préciser CR ou LF ou CR/LF)


Chaque instruction reçoit une réponse contenant au minimum un code d'erreur comme premier paramètre.

Une réponse peut être constituéée de un ou plusieurs paramètres séparés par un pipe "|" :

  • Paramètre n°1 : un code d'erreur sous forme d'une valeur numérique entière (l'énuméré est disponible dans le fichier Instruction.h)
  • Paramètre n°2 : si besoin en fonction de l'instruction
  • Paramètre n°xxx : ...

Les instructions avec les valeurs retournées sont détaillées ci-dessous.

En cas de problème interne durant l'exécution (détection d'un pointeur null par exemple), une instruction peut renvoyer le code d'erreur : INTERNAL_ERROR


Dans le cas d'une commande contenant plusieurs instructions, la réponse globale est composée d'une réponse à chaque instruction séparé par un retour chariot.


Les instruction sont sensibles à la casse.


La liste des instructions possibles est :

  • help() : fournit la liste de l'API avec des exmples d'utilisation
  • Retour de l'instruction :
  • La fonction s'est exécutée avec succès (toujours le cas):

> Paramètre n°1 : 0 Code d'erreur OK


  • helpInstruction(<nomInstruction>) : fournit de l'aide sur une instruction donnée
  • Retour de l'instruction :
  • Cas où la fonction s'est exécutée avec succès :

> Paramètre n°1 : INSTRUCTION_OK

  • Cas où l'instruction <nomInstruction> n'existe pas :

> Paramètre n°1 : INVALID_ARGUMENT

> Paramètre n°2 : rappel de l'instruction complète

> Paramètre n°3 : le nom de l'instruction inexistante


  • setData(<nomData>, valeur>) : écrit "valeur" dans la donnée <nomData>
  • Retour de l'instruction :
  • Cas où la fonction s'est exécutée avec succès :

> Paramètre n°1 : INSTRUCTION_OK

  • Cas où la donnée <nomData> n'existe pas dans le DataManager:

> Paramètre n°1 : INVALID_ARGUMENT

> Paramètre n°2 : rappel de l'instruction complète

> Paramètre n°3 : le nom de la donnée non trouvée


  • getData(<nomData>) : récupère la valeur de la donnée <nomData>
  • Retour de l'instruction :
  • Cas où la fonction s'est exécutée avec succès :

> Paramètre n°1 : INSTRUCTION_OK

> Paramètre n°2 : valeur de la donnée

  • Cas où la donnée <nomData> n'existe pas dans le DataManager:

> Paramètre n°1 : INVALID_ARGUMENT

> Paramètre n°2 : rappel de l'instruction complète

> Paramètre n°3 : le nom de la donnée non trouvée


  • getDatas() : récupère la valeur de toutes les données du DataManager
  • Retour de l'instruction :
  • Cas où la fonction s'est exécutée avec succès :

> Paramètre n°1 : INSTRUCTION_OK

> Paramètre n°2 : Les valeurs de toutes les données au format donnee=valeur. Les données sont séparées par un retour chariot.


  • createData(<nomData>, valeur> : crée la donnée <nomData> et l'initialise à "valeur".
  • Retour de l'instruction :
  • Cas où la fonction s'est exécutée avec succès :

> Paramètre n°1 : INSTRUCTION_OK

  • Cas où la donnée <nomData> existe déjà dans le DataManager:

> Paramètre n°1 : INVALID_ARGUMENT

> Paramètre n°2 : rappel de l'instruction complète

> Paramètre n°3 : le nom de la donnée déjà existante


Les codes d'erreurs :

  • 0 (INSTRUCTION_OK): OK, la commande a été effectuée avec succès
  • 1 (INVALID_INSTRUCTION): l'instruction est inconnue
  • 2 (INVALID_ARGUMENT) : problème avec un des arguments passés à l'instruction
  • ...

Voir l'énuméré dans le fichier Instruction.h


Exemples :

Instruction : getData(PositionX)

Retour : 0|3.25


Instruction : Toto

Retour : 1


Instruction : createData(newData, 123.456)

Retour : 0


Instruction : helpInstruction(instructionInexistante)

Retour : 2|helpInstruction(instructionInexistante)|instructionInexistante
