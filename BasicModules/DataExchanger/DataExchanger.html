DataExchanger

Par Nico


Description générale :


DataExchanger est un module permettant d'échanger des données du DataManager entre 2 applications basées sur une architecture Labotbox (transfert bi-directionnel et simultané -> "fullduplex").


Le mécanisme est basé sur un client-serveur TCP.

Une des 2 applications doit être configurée en client et l'autre en serveur.

Les 2 application peuvent tourner sur un même PC ou sur des PC différents.

Le module DataExchanger peut être configuré en client ou en serveur.


Tout ce que peut faire le serveur est faisable par le client et inversement. Il n'y a aucune restriction de fonctionnalité du fait d'une configuration en client ou en serveur. Le choix de l'application qui sera le serveur ou le client est purement arbitraire.

L'application locale peut demander à l'application distante des services par des requêtes (commencer ou arrêter le transfert, ajouter ou supprimer une donnée au transfert).

Tant que la connexion n'est pas établie entre le client et le serveur, les opérations principales du module sont inhibées (activation du transfert, requêtes vers le distant, ...).


Si une donnée reçue en provenance du distant n'existe pas encore dans le DataManager local, elle est automatiquement créé.


Lorsqu'une donnée est indiquée comme "à transférer", chaque écriture dans cette donnée provoquera automatiquement son transfert vers le distant (que la donnée ait changé de valeur ou non).

ATTENTION : pour cette raison, si une même donnée est indiquée à transférer dans un sens ET dans l'autre, il y a un risque d'instabilité des 2 applications (l'application no1 recevant la donnée X va la renvoyer immédiatement à l'application no2 qui elle même va la renvoyer immédiatement à l'application no1, qui elle même va la renvoyer à l'application no2, ....).


Les paramètres de configuration :

  • exchanger_type : choix du type. Peut prendre les valeurs client ou server
  • exchanger_port : numéro du port de communication (utilisé pour le client ou le serveur)
  • exchanger_ip : adresse IP du serveur (utilisé pour le client uniquement). Si les 2 applications tournent sur le même PC, l'adresse IP à utiliser est 127.0.0.1
  • autoreconnect : active les tentatives de reconnexions automatiques en cas de déconnexion (utilisé pour le client uniquement). Peut prendre les valeurs true ou false.


Pricinpe d'utilisation :

Le bandeau principal du module affiche DataExchanger<XXX> où <XXX> représente si le module est configuré en client ou en serveur.



La LED "Connexion" indique si le client et le serveur sont connectés (allumée = connexion établie).


Le bandeau du bas indique pour le serveur le numéro du port sur lequel le serveur est en attente de connexion du client.


Lors du lancement de l'application, le serveur se crée automatiquement et le client tente de se connecter automatiquement au serveur.

Si le paramètre "autoreconnect" est à "true", Il n'y a pas d'ordre de lancement des applications client ou serveur. Le client peut être lancé avant ou après le serveur.

Dès que la connexion est établie entre le client et le serveur, le transfert dans les 2 sens est activé.


Le module est décomposé en 2 onglets :

  • local : permet de configurer le module DataExchanger de l'application locale (liste des données à transférer vers le distant, activer ou arrêter le transfert vers le distant, charger une liste de données à transférer vers le distant, ...)
  • distant : permet de configurer le module DataExchanger de l'application distante (demande d'activation ou arrêt du transfert, ajout ou suppression d'une donnée à transférer)

Appuyer sur le bouton "Start Request" de l'application locale conduit au même résultat que d'activer la checkbox "On/Off" sur l'application distante.

De la même manière, appuyer sur le bouton "Stop Request" de l'application locale conduit au même résultat que décocher la checkbox "On/Off" sur l'application distante.
