DataPlayer

Par Nico


Description générale :


DataPlayer est un module permettant de jouer ou rejouer des signaux sur les data du DataManager.


Fonction "Générateur de signaux"


2 formats de fichiers d'entrés sont acceptés par l'outil

Format à 1 colonne

  • La 1ère ligne contient une description
  • 1 ligne par échantillon

Dans ce cas, la durée d'un échantillon sera précisée sur l'IHM

Format à 2 colonnes

  • La 1ère ligne contient une description
  • 1 ligne par échantillon
  • La première colonne représente la valeur de la data
  • La 2ème colonne représente la durée de l'échantillon en [msec]

Fonctionnement :

  1. Sélectionner une donnée et sélectionner le fichier contenant le signal à lui appliquer.
  2. Sélectionner la durée de l'échantillon si elle n'est pas contenue dans le fichier d'entrée.
  3. Sélectionner le nombre de cycle à jouer (valeur spéciale "-1" pour que le signal soit joué en boucle infine).
  4. Démarrer ou arrêter la génération du signal (lorsque le générateur est arrêté, la donnée conserve sa dernière valeur).

Lorsqu'une donnée sélectionnée est pointée par le curseur, les paramètres du générateur sont mis à jour (nom du signal appliqué, durée de l'échantillon, nombre de cycles)

Plusieurs générateurs peuvent être lancés sur plusieurs data du DataManager. Les générateurs sont indépendants.







Fonction "Rejoueur de trace"


Le rejoueur de trace permet de rejouer un fichier enregistré et s'applique à une ou plusieurs données du DataManager.

Plusieurs formats sont acceptés.


Fichier au format "Trace"

Ce format est celui enregistré par le module DataView.

Format à 3 colonnes

  • La première ligne contient la description des colonnes
  • La première colonne représente le timestamp en [sec]
  • La 2ème représente le nom de la data
  • La 3ème représente la valeur de la data pour ce timestamp


Plusieurs lignes peuvent avoir le même timestamp. Dans ce cas, les données d'un même timestamp seront jouées en même temps.

Dans ce format, la première colonne représente le temps.

Ensuite, chaque colonne représente une Data.

La première ligne contient la description.

Les autres lignes correspondent aux valeurs à donner à chaque Data pour ce step.


Le temps peut être un timestamp ou la durée du step

Le temps peut être exprimé en secondes ou en millisecondes.

Les mots clés [sec] ou en [msec] présents dans le nom de la première colonne indiquent l'unité.

Les mots clés Duration ou Timestamp présents dans le nom de la première colonne indiquent si la colonne représente un temps ou la durée du step.

Fonctionnement :

  1. Créer un player si besoin (le premier player "Player1" est créé par défaut).
  2. Sélectionner le fichier d'entrée (le format et l'unité de temps seront détectés automatiquement à partir du nom des colonnes)
  3. Surcharger si besoin la durée du step par une valeur en msec. Dans ce cas, le temps du fichier sera ignoré.
  4. Sélectionner le nombre de cycle à jouer (valeur spéciale "-1" pour que le signal soit joué en boucle infine).

Il est possible de créer plusieurs players. Dans ce cas, un fichier spécifique sera associé à chaque player. Il est possible dans ce cas de synchroniser le démarrage ou l'arrêt de tous les players.

Déplacer manuellement le curseur ou utiliser les boutons step up et step down permet de faire défiler manuellement les steps.
