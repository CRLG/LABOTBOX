RaspiGPIO

Par Nico


RaspiGPIO est un module permettant de contrôler les GPIO du Raspberry Pi.

Le module exploite la librairie wiringPi. Par conséquence, le module n'est compilable que sur plateforme Raspberri Pi. Une option dans le fichier .pro permet d'activer ou non la compilation du module : CONFIG += RASPBERRY_PI


La numérotation des GPIO utilise la notation BCM.

Par défaut, tous les GPIO sont dans l'état "U" : Unused.


GPIO numériques :

Chaque GPIO peut être :

  • Configuré en :
  • Entrée tout-ou-rien
  • Entrée avec pull up
  • Entrée avec pull down
  • Sortie tout-ou-rien
  • Sortie PWM (!! implémentation non terminée)
  • Lu : dès qu'un GPIO est configuré (en entrée ou sortie), la lecture périodique de l'état de la pin est activée. La LED associée représente la lecture du port.
  • Ecrit : lorsqu'un GPIO est configuré en sortie, une valeur (0 ou 1) peut être écrite sur le port.


- Configuration des GPIO depuis le fichier de paramétrage EEPROM.ini

Chaque GPIO peut être configuré en entrée ou sortie depuis le fichier de paramètrage EEPROM.ini. Ainsi, au démarrage de l'application, le GPIO sera configuré sans aucune autre action nécessaire.

La syntaxe dans le fichier EEPROM.ini est : gpio_<numero>=<config>

Pour connaitre les valeurs possibles de <numero>, se reporter au mapping des GPIO du Rabspberry ou à l'interface utilisateur du module.

Rappel : La numérotation des GPIO utilise la notation BCM.

Les valeurs possibles de <config> sont :

  • input
  • input pull up
  • input pull down
  • output
  • pwm output


Exemples :

gpio_16=Input

gpio_17=input pull up

gpio_21=input pull Down

gpio_26=OUTPUT

gpio_27=output


- Manipulation des GPIO depuis l'IHM :

La configuration, la lecture et l'écriture des GPIO peut se faire depuis l'interface utilisateur du module.

L'interface utilisateur représente le connecteur 40 pins du Raspberry.

Etant donné le nombre de GPIO, l'interface utilisateur est condensée. On y retrouve le numéro de pin du connecteur, l'état du port, la configuration du GPIO, son état et la possibilité de forcer l'état si le GPIO est configuré en sortie.

Les LED correspondent au retour d'état du GPIO, tandis que les checkbox permettent l'écriture.

  • La notation "U" est utilisé pour indiquer que le GPIO n'est pas utilisé : "Unused" (c'est la configuration par défaut)
  • La notation "I" est utilisé pour indiquer que le GPIO est configuré en "Input"
  • La notation "Id" est utilisé pour indiquer que le GPIO est configuré en "Input Pull Down"
  • La notation "Iu" est utilisé pour indiquer que le GPIO est configuré en "Input Pull Up"
  • La notation "O" est utilisé pour indiquer que le GPIO est configuré en "Output"
  • La notation "P" est utilisé pour indiquer que le GPIO est configuré en "PWM Output"



- Manipulationd des GPIO depuis les variables du DataManager

Une fois configurés, les GPIO sont contrôlables depuis des variables du DataManager.

A chaque GPIO configuré en entrée ou en sortie, une variable RaspiGPIO_<numero> est automatiquement créée dans le DataManager. Cette variable mise à jour périodiquement par le module et reflète l'état du GPIO suite à relecture.


De plus, à chaque GPIO configuré en sortie, une 2ème variable RaspiOutGPIO_<numero> est automatiquement créee dans le DataManager. Lorsqu'une opération d'écriture est réalisée dans cette variable, l'écrture de la valeur correspondante est effectuée sur le GPIO.


Ainsi, pour un GPIO configuré en sortie, 2 variables seront crées :

  • Une pour permette l'écriture
  • Une reflétant l'état réel du port


- Manipulation des GPIO depuis l'API :

La configuration, lecture, écriture des GPIO est également disponible sous forme d'une API mise à disposition par le module.

  • void configPinMode(unsigned int gpio_num, QString mode);
  • void setPinModeInput(unsigned int gpio_num, unsigned int pullup_down);
  • void setPinModeOutput(unsigned int gpio_num, unsigned int init_state);
  • void setPinModePWMOutput(unsigned int gpio_num, unsigned int init_pwm);
  • bool readPin(unsigned int gpio_num);
  • void writePin(unsigned int gpio_num, bool state);
  • void writePwmPin(unsigned int gpio_num, float percent);
