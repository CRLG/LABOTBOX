Ce module permet de prototyper des stratégies, les tester directement sur le robot puis de générer le code compatible avec Modélia une fois le prototypage terminé.


La fenêtre est composée de 2 zones:



La zone de Gauche est dédiée à la création, la manipulation et le test des stratégies.

La zone de Droite est le stockage par onglet des stratégies. Les contrôles de la zone de gauche s'appliquent, sauf rares exceptions, à l'onglet de stratégie sélectionné.


CONSTRUIRE UNE STRATEGIE

Principe global

Il y a des actions et des transitions.


Les actions

Les actions sont composées par:

  1. Le pilotage des servomoteurs SD20 (type futaba)
  2. Le pilotage en position ou en vitesse des servomoteurs Dynamixel AX
  3. Le pilotage des moteurs (y compris ceux des roues motrices du robot)
  4. Le pilotage des sorties de PowerElectroBot (relais)
  5. Les commandes d'asservissement en position du robot ou en position du rack mobile
  6. Les actions libres (elle ne peuvent être testées et servent uniquement à combler les potentiels manques du générateur de code)
  7. Les actions vides (les noeuds) utilisés pour les transitions multiples (covergentes ou divergentes)


Les transitions

Les transitions sont composées par:

  1. Les temporisations en ms
  2. Les convergences d'asservissement (elle possèdent une temporisation intégrée)
  3. Les tests "inférieur", "égal" ou "supérieur" sur n'importe quelle variable du DataManager
  4. Les transitions libres (elle ne peuvent être testées et servent uniquement à combler les potentiels manques du générateur de code)

Les commandes

  • :le bouton "plus" permet d'ajouter l'action ou la transition à la stratégie courante (onglet sélectionné)
  • :le bouton "play" permet de jouer les actions directement sur le robot
  • : le bouton "stop" permet d'arrêter une action en cours (exemple un moteur en train de tourner)


Détail pour chaque action/transition


Les servomoteurs SD20



On choisit l'identifiant (ou le nom s'il est renseigné dans le fichier DataCenter.ini) ainsi que la valeur à attribuer (entre 0 et 254).


Les servomoteurs AX


On choisit l'identifiant (ou le nom s'il est renseigné dans le fichier DataCenter.ini) ainsi que la valeur à attribuer (entre 1 et 1024).

La valeur peut être une consigne de position ou de vitesse).


Les moteurs


On choisit l'identifiant (ou le nom s'il est renseigné dans le fichier DataCenter.ini) ainsi que la valeur à attribuer (entre 0 et 100).


PowerElectroBot


On choisit l'identifiant (ou le nom s'il est renseigné dans le fichier DataCenter.ini) ainsi que la valeur à attribuer (soit 1, soit 0).


L'asservissement


On choisi le mode d'asservissement:

  • XY, XYTeta et DistanceAngle pour l'asservissement en position du robot
  • position pour l'asservissement ascenseur

Les paramètres sont rendus accessibles suivant le mode d'asserbissement choisi.

Le bouton geData permet de récupérer les coordonnées actuelles du robot.


Les actions libres


C'est un texte libre qui sera retranscrit tel quel lors de la génération de code, il ne sera bien entendu pas jouable directement sur le robot.

Les boutons flèches permettent de récupérer ou écraser un texte d'une action libre déjà existante et sélectionnée dans la séquence courante.


Les Noeuds


C'est soit un noeud de convergence pour des transitions amonts, soit une aiguillage vers des transitions avals.

Un noeud est forcément nommé, si le nom n'est pas renseigné avant l'ajout dans la séquence alors un nom automatique sera attribué.

Les tests


Condition sur n'importe quelle variable du data center. Choisir la variable dans la liste déroulante, l'égalité à tester (inférieur, égal ou supérieur) et la valeur de test.


Les transitions libres


C'est un texte libre qui sera retranscrit tel quel lors de la génération de code, il ne sera bien entendu pas jouable directement sur le robot.

Les boutons flèches permettent de récupérer ou écraser un texte d'une transition libre déjà existante et sélectionnée dans la séquence courante.
